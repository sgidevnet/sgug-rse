DIDBSNG Missing Functionality

The following have been identified as "missing" for building some of the packages listed below. Possibly either they get added to regular didbs (to make sure they work) and/or they are just introduced as SRPM/RPM packages.

  • chrpath
  • ldconfig_scriplets
  • redhat_rpm_config (some packages use macro invocations to turn libtool launchers into re-linked executables) - this also might be where debug info is extracted into separate packages

DIDBSNG Package Progress

A list of the packages we are trying to get building under an "RPM" based environment and their status.

  • SRPM/SPEC Found =

    Found existing SRPM/SPEC file for it

  • SPEC Builds OK = 
    rpmbuild -ba BLAH.spec --nocheck
  • First RPMs Install OK = 
    rpm -ivh BLAH-version.didbsng.mips.rpm
  • SPEC DIDBS Patched =

    Checked if didbs patches moved over

  • didbsngshell OK? = 
    didbsngshell
    /usr/didbsng/bin/blah
  • didbsngshell Builds OK?

    (launch didbsngshell, build using gcc in there)

  • Active Deps OK?

    (put back original deps, does it still build OK within didbsngshell?)

  • Notes/make check

    A place to note blocking issues / glaring problems
